# Quick Task 260412-vl1: Consolidate yarn clean commands

## Context

The yarn palette currently has three separate clean commands:
- `a` → `RnCleanAndroid`
- `c` → `RnCleanCocoapods`
- `n` → `RmNodeModules`

The `ModalState::CleanToggle` already exists with full toggle UX (node_modules / pods / android / sync_after + confirm), but no key opens it. The intent was a single "clean" entry under yarn that opens this modal so the user can pick what to clean in one place.

## Scope

Replace the three separate yarn palette keys with a single `c` entry that opens the `CleanToggle` modal.

## Plan

### Task 1: Wire a single `c` yarn entry to open CleanToggle modal

**Files:**
- `src/action.rs` — add `Action::OpenCleanMenu` variant
- `src/app.rs` — handle `OpenCleanMenu` in update(); remap yarn palette `c` to it; remove `a`/`c`/`n` individual clean entries in Yarn palette
- `src/ui/footer.rs` — replace yarn palette hints `clean android` / `clean cocoapods` / `rm node_modules` with single `("c", "clean…")`
- `src/ui/help_overlay.rs` — replace the three yarn clean rows with a single `c → Clean… (select targets)` row

**Action:**
1. Add `OpenCleanMenu` to `Action` enum in `src/action.rs`.
2. In `src/app.rs` key-mapping for `PaletteMode::Yarn`, replace the `a`/`c`/`n` arms with a single `Char('c') => Some(Action::OpenCleanMenu)`.
3. In the update() function, handle `Action::OpenCleanMenu` by:
   - Clearing `palette_mode`
   - Setting `state.modal = Some(ModalState::CleanToggle { options: CleanOptions::default() })`
4. Update footer yarn hints: drop the three clean entries, add one `("c", "clean…")`.
5. Update help_overlay yarn section accordingly.

**Verify:**
- `cargo check --all-targets`
- `cargo test` passes
- Manual path: `y → c` opens clean modal; `n/p/a` toggles; `i` toggles sync_after; `x`/Enter confirms.

**Done:**
- Single `c` yarn entry opens CleanToggle modal
- No separate yarn-palette clean commands remain
- Footer and help overlay reflect the new entry
