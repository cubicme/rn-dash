  <name>Task 2: Fix metro conflict detection during app-managed worktree switch</name>

Fix UAT gap test 9: Enter on worktree with app-managed metro shows external conflict prompt instead of just switching.

**Root cause:** In the `Action::MetroStart` handler (around line 580), external metro detection always runs via `detect_external_metro(8081)`. When doing a worktree switch (Enter key), the flow is: WorktreeSwitchToSelected -> MetroStop -> (MetroExited) -> MetroStart. The old metro process may still be releasing port 8081 when the new MetroStart fires, so `detect_external_metro` finds it and shows the ExternalMetroConflict modal.

**Fix:** In the `Action::MetroStart` handler, after the `is_running()` early return for restart, add a check: if `state.pending_restart` is true OR `state.pending_switch_path.is_some()`, skip external metro detection and go directly to `MetroStartConfirmed`. The rationale: if we just stopped our own metro as part of a switch/restart, any lingering port usage is from our own process, not external.

Specifically, change the MetroStart handler from:
```rust
Action::MetroStart => {
    state.palette_mode = None;
    if state.metro.is_running() {
        state.pending_restart = true;
        update(state, Action::MetroStop, metro_tx, handle_tx);
        return;
    }
    // Check for external metro conflict before spawning
    let tx = metro_tx.clone();
    tokio::spawn(async move {
        if let Some(info) = crate::infra::port::detect_external_metro(8081).await {
            let _ = tx.send(Action::ExternalMetroDetected(info));
        } else {
            let _ = tx.send(Action::MetroStartConfirmed);
        }
    });
}
```

To:
```rust
Action::MetroStart => {
    state.palette_mode = None;
    if state.metro.is_running() {
        state.pending_restart = true;
        update(state, Action::MetroStop, metro_tx, handle_tx);
        return;
    }
    // Skip external detection when restarting our own metro (worktree switch or manual restart).
    // The port may still be releasing from our just-killed process — not an external conflict.
    if state.pending_switch_path.is_some() {
        let _ = metro_tx.send(Action::MetroStartConfirmed);
        return;
    }
    // Check for external metro conflict before spawning
    let tx = metro_tx.clone();
    tokio::spawn(async move {
        if let Some(info) = crate::infra::port::detect_external_metro(8081).await {
            let _ = tx.send(Action::ExternalMetroDetected(info));
        } else {
            let _ = tx.send(Action::MetroStartConfirmed);
        }
    });
}
```

Note: We check `pending_switch_path` specifically (not `pending_restart`) because by the time MetroStart fires from MetroExited, `pending_restart` has already been set back to false (line 649). But `pending_switch_path` is consumed via `.take()` only in the MetroExited handler BEFORE the MetroStart call... wait, let me re-read.

Actually, looking at MetroExited (line 648-654):
```rust
if state.pending_restart {
    state.pending_restart = false;
    if let Some(path) = state.pending_switch_path.take() {
        state.active_worktree_path = Some(path);
    }
    update(state, Action::MetroStart, metro_tx, handle_tx);
}
```

Both `pending_restart` and `pending_switch_path` are consumed BEFORE MetroStart is called. So neither is available when MetroStart runs.

**Revised fix:** Add a new boolean field `skip_external_check: bool` to AppState (default false). Set it to true in MetroExited when `pending_restart` was true, right before calling MetroStart. Then in MetroStart, check and consume this flag:

In AppState struct definition, add:
```rust
pub skip_external_metro_check: bool,
```
Initialize to `false` in AppState::new() or wherever AppState is constructed.

In MetroExited handler, before the `update(state, Action::MetroStart, ...)` call:
```rust
state.skip_external_metro_check = true;
```

In MetroStart handler, after the `is_running()` check, add:
```rust
if state.skip_external_metro_check {
    state.skip_external_metro_check = false;
    let _ = metro_tx.send(Action::MetroStartConfirmed);
    return;
}
```
