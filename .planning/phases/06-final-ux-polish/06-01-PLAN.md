Add a private function `should_suppress_metro_line(line: &str) -> bool` near the `stream_metro_logs` function (around line 1742 in app.rs). The function should return true for lines that should be dropped:

```rust
fn should_suppress_metro_line(line: &str) -> bool {
    // Watchman warnings — primary noise source
    if line.contains("watchman warning") || line.contains("watchman:") {
        return true;
    }
    // Metro startup banner decoration — deeply indented whitespace-only lines
    let trimmed = line.trim();
    if trimmed.is_empty() {
        return true;
    }
    false
}
```

Then modify `stream_metro_logs` to call this filter before sending `MetroLogLine`. In the two `Ok(Some(l))` arms (stdout and stderr, around lines 1757 and 1763), wrap the send in:

```rust
Ok(Some(l)) => {
    if !should_suppress_metro_line(&l) {
        let _ = tx.send(Action::MetroLogLine(l));
    }
}
```

Apply this pattern to BOTH the stdout and stderr reader arms.

Start conservative with filtering patterns (only watchman and empty lines). Do NOT filter lines containing just "warn" — that would suppress legitimate build warnings.
