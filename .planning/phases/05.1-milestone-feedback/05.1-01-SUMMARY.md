- `src/ui/panels.rs` - Updated render_command_output to use crate::app::active_output() and active_output_scroll() instead of state.command_output/command_output_scroll

## Decisions Made

- Output persists per worktree across command runs — dispatch_command appends separator (not clears) per CONTEXT.md design
- CommandCancel is all-or-nothing: clears entire command_queue alongside aborting current task
- WORK-06 lazy install uses queue push instead of `pending_command_after_install`: enqueue run command, dispatch yarn install, CommandExited pops original run command from queue
- Zero-cost empty default for active_output() uses `std::sync::LazyLock<VecDeque<String>>`
- Per-worktree output caps at MAX_COMMAND_LINES (1000) per worktree via while-pop_front loop in dispatch_command
