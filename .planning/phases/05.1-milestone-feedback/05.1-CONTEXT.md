<decisions>
## Implementation Decisions

### Worktree Table Design
- Move worktree list to bottom of the layout (few worktrees, lots of info per row)
- Table columns in order: Label | Worktree name | Branch | Ticket # | Ticket title | Status icons
- Status icon columns are fixed-width without headers — icons appear/disappear in constant-width cells
- Icon legend displayed in footer bar alongside keybinding hints (e.g., "●=metro ⚠=stale")
- Metro-active worktree: pinned to top of list + highlighted row (bold/background) + ● icon — all three, can remove later
- No sorting — natural discovery order from filesystem
- Columns truncate with … on narrow terminals — TITLE and BRANCH truncate first, then LABEL if still tight
- Max ~10 worktrees (currently 4) — always fully visible, no virtualization needed

### Command Scheme (complete remap)
Top-level keys from worktree list:
- `a` → Android submenu
  - `d` → run-android
  - `e` → run-android --list-devices (interactive device prompt — redirect output for selection)
  - `r` → release build: gradlew assembleRelease + adb install (Claude's discretion on UX, no cd needed — dashboard controls cwd)
- `i` → iOS submenu
  - `d` → device (run-ios on physical device)
  - `e` → simulator (sort by recent usage + type-to-filter when typing in picker)
  - `p` → yarn pod-install
- `x` → Clean submenu (toggle options, then confirm)
  - `n` → toggle node_modules
  - `p` → toggle pods
  - `a` → toggle android
  - `i` → toggle sync-after-clean (yarn + pods)
  - `x` or Enter → confirm and execute selected cleans
- `s` → Sync submenu
  - `i` → yarn install
  - `u` → unit-tests
  - `t` → check-types --incremental
  - `j` → jest [filter]
  - `l` → lint --quiet --fix
- `g` → Git submenu
  - `f` → fetch --all --tags
  - `p` → pull
  - `P` → push
  - `X` → reset hard (auto: fetch first, then reset to origin/same-branch — no generic reset)
  - `b` → checkout
  - `c` → checkout -b
  - `r` → rebase
- `C` (Shift-c) → Open Claude Code in multiplexer tab
- `L` → Set label on selected worktree
- `f` → Toggle fullscreen for focused pane (log, metro, command output)
- `!` → Run shell command in worktree directory
- `Enter` → Switch metro to selected worktree (existing)

### Command Queue
- Users can queue multiple commands — they execute sequentially in FIFO order
- Queue is visible (what's pending) and clearable (all-or-nothing, no individual item removal)
- Always show the currently running command prominently
- Cancel currently running command supported (existing CommandCancel action)

### Output Persistence
- Command output persists per worktree directory
- Switching worktrees shows that worktree's output history
- Output panel retains content across command runs (not cleared on new command unless user clears)

### Log Panel
- `x` key clears the log panel content (when log panel focused)
- Note: `x` at top-level opens clean submenu, `x` in log panel clears — context-sensitive

### Sync-Before-Run Behavior
- When user triggers a run command (run-android, run-ios) on a stale worktree, show a prompt: "Worktree is stale. Sync before running?"
- Staleness checks both: node_modules (package.json/yarn.lock mtime) AND Pods/ (Podfile.lock mtime, for iOS runs)
- If user says Yes: queue sync commands (yarn install, pod-install if iOS) followed by the run command — all via command queue
- No "Always" option — prompt each time

### Simulator Picker Enhancement
- Sort simulators by time of last use (most recent first)
- Type-to-filter: start typing to narrow simulator list in real-time
- Only for simulator picker (i>e) — device pickers (a>e, i>d) stay as-is (usually 1-2 devices)

### Multiplexer Abstraction
- Trait with two methods: `new_window(path, name, command)` and `is_available()` — possibly `rename_window()`
- Two adapters: TmuxAdapter and ZellijAdapter
- Auto-detect: check $TMUX first, then $ZELLIJ. If both set, prefer tmux
- Degrade gracefully without multiplexer — Claude Code and multiplexer features disabled with message, everything else works
- No splits, send-keys, or pane management — just window creation and detection

### Config: Claude Command Flags
- Add a config field for Claude Code command flags (e.g., `--dangerously-skip-permissions`)
- Current config should include `--dangerously-skip-permissions` as default flag

### Claude's Discretion
- Release build UX (single command vs split steps for a>r)
- Exact command queue UI placement and visual treatment
- Log clear key behavior when log panel is not focused
- Output persistence storage format and limits
- Fullscreen toggle animation/transition
- Shell command input modal design (for !)

</decisions>

<specifics>
## Specific Ideas

- Clean submenu (`x`) uses a toggle pattern — user checks/unchecks options (n, p, a, i) then confirms with `x` or Enter
- Git reset hard (`g>X`) should always fetch first, then reset to origin/same-branch — not a generic reset
- "I couldn't find in the codebase where the order and dependency of actions are coded" — v1 uses ad-hoc `pending_restart` bool and inline stale checks. Phase 5.1 replaces this with a proper command queue.
- Simulator sort-by-recent implies tracking last-used simulator somewhere (config or cache file)

</specifics>
