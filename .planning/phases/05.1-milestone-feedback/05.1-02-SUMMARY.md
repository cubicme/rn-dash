# Phase 05.1 Plan 02: Multiplexer Abstraction and Claude Flags Summary

Trait-based multiplexer abstraction with TmuxAdapter and ZellijAdapter, plus DashConfig.claude_flags field defaulting to --dangerously-skip-permissions.

## What Was Built

### Multiplexer Trait (`src/infra/multiplexer.rs`)

New module providing a trait-based terminal multiplexer abstraction:

- `Multiplexer` trait with `new_window(path, name, command)` and `is_available()` methods, requiring `Send + Sync + Debug` supertraits
- `TmuxAdapter`: wraps `tmux new-window -d` CLI, checks `$TMUX` env var for availability
- `ZellijAdapter`: uses `zellij action new-tab` + `write-chars` + `go-to-previous-tab` to emulate background tab creation, checks `$ZELLIJ` env var
- `detect_multiplexer()`: returns `Option<Box<dyn Multiplexer>>`, checking `$TMUX` first then `$ZELLIJ`

### Infra Module Registration (`src/infra/mod.rs`)

Added `pub mod multiplexer;` to the module list.

### Deprecation Notice (`src/infra/tmux.rs`)

Added deprecation doc comment to `open_claude_in_worktree` — no code changes, function kept until app.rs is rewired in a later plan.

### DashConfig Extension (`src/infra/config.rs`)

Added `claude_flags: String` field with `#[serde(default = "default_claude_flags")]` so existing config files without the field get `--dangerously-skip-permissions` automatically.

## Decisions Made

- **Multiplexer supertrait bounds**: `Send + Sync + Debug` required so the trait object can live in `AppState` (which requires `Send + Sync`) and be introspected in error output.
- **ZellijAdapter focus handling**: Zellij's `new-tab` steals focus unlike `tmux new-window -d`. Added `go-to-previous-tab` call after `write-chars` to restore focus to the dashboard. Non-fatal — failure logged as warning only.
- **Priority in detect_multiplexer()**: `$TMUX` checked before `$ZELLIJ` since a user can be in a tmux session inside Zellij. Most specific environment wins.
- **tmux.rs unchanged**: `open_claude_in_worktree` kept as-is with a deprecation comment. Removing it now would break app.rs — rewire is deferred to a later plan.
- **claude_flags serde default**: Uses the established `default_*` function pattern already present in config.rs (`default_auth_mode`). Existing config files without the field get the user-chosen default automatically on next load.
