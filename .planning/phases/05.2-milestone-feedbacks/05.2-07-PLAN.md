Fix scrollbar position mismatch when pressing G (UAT Test 7).

The issue: When G is pressed (ScrollToBottom), the metro log auto_follow is set to true. In render_metro_pane, when auto_follow is true, the `scroll` variable is computed as `lines.len().saturating_sub(visible_height)`. The scrollbar uses `ScrollbarState::new(lines.len()).position(scroll)`.

The problem is that ScrollbarState::new(content_length) with .position(scroll) expects position to be in range [0, content_length]. When auto_follow computes `lines.len() - visible_height`, this is the correct TEXT scroll offset but the scrollbar's content_length should be `lines.len().saturating_sub(visible_height)` (the scrollable range), not `lines.len()` (total content).

Fix in render_metro_pane AND render_log_panel AND render_command_output: change scrollbar initialization to:
```rust
if lines.len() > visible_height {
    let max_scroll = lines.len() - visible_height;
    let mut scrollbar_state = ScrollbarState::new(max_scroll).position(scroll.min(max_scroll));
    // render scrollbar...
}
```

This ensures the scrollbar range matches the actual scrollable range, and when scroll is at `lines.len() - visible_height` (bottom), the scrollbar shows at the bottom.

Apply this fix to all three pane rendering functions that have scrollbars: render_metro_pane, render_log_panel, render_command_output.
