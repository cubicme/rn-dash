2. **src/infra/config.rs**: Create module with:
   - `use crate::infra::labels::config_dir;` to reuse the shared config directory.
   - `DashConfig` struct with fields: `jira_base_url: String`, `jira_email: Option<String>` (with `#[serde(default)]`), `jira_token: String`, `auth_mode: String` (with `#[serde(default = "default_auth_mode")]` defaulting to `"cloud"`). Derive `Debug, Deserialize, Serialize`.
   - `load_config() -> anyhow::Result<Option<DashConfig>>`: Read `config_dir().join("config.json")`. Return `Ok(None)` on `NotFound`, propagate other errors. Same pattern as `load_labels()`.
   - `save_config(config: &DashConfig) -> anyhow::Result<()>`: Write pretty JSON to `config.json`, then immediately set permissions to 0600 via `std::os::unix::fs::PermissionsExt::from_mode(0o600)`. Call `create_dir_all` on config_dir first. Add `#[cfg(unix)]` guard on the permissions call with a no-op `#[cfg(not(unix))]` fallback.

3. **src/infra/jira_cache.rs**: Create module with:
   - `use crate::infra::labels::config_dir;`
   - `pub fn cache_path() -> PathBuf { config_dir().join("jira_cache.json") }`
   - `pub fn load_jira_cache() -> anyhow::Result<HashMap<String, String>>`: Same pattern as `load_labels()` — returns empty HashMap on NotFound.
   - `pub fn save_jira_cache(cache: &HashMap<String, String>) -> anyhow::Result<()>`: Write pretty JSON. No 0600 permissions needed (titles are not credentials). Call `create_dir_all` first.

4. **src/infra/jira.rs**: Create module with:
   - `JiraClient` trait (behind `#[async_trait]`): single method `async fn fetch_title(&self, ticket_key: &str) -> Option<String>`. Trait bound: `Send + Sync`.
   - `HttpJiraClient` struct with fields: `client: reqwest::Client`, `base_url: String`, `auth_mode: String`, `email: Option<String>`, `token: String`.
   - `HttpJiraClient::new(config: &DashConfig) -> anyhow::Result<Self>`: Build a `reqwest::Client` (no default auth headers — auth applied per-request for clarity).
   - `impl JiraClient for HttpJiraClient`: `fetch_title` builds URL `{base_url}/rest/api/3/issue/{ticket_key}?fields=summary`. If `auth_mode == "datacenter"`, uses `.bearer_auth(&self.token)`. Otherwise uses `.basic_auth(self.email.as_deref().unwrap_or(""), Some(&self.token))`. Parse response as `serde_json::Value`, extract `resp["fields"]["summary"].as_str()`. Return `None` on any error (`.ok()?` chain) — never panic, never surface errors to user.
   - `pub fn extract_jira_key(branch: &str) -> Option<String>`: Pure function. Split branch by `/`, then for each segment split by `-` using `splitn(3, '-')`. If first part is `"UMP"` and second part is non-empty all-ASCII-digits, return `Some(format!("UMP-{}", digits))`. No regex crate needed.
   - `pub fn is_inside_tmux() -> bool { std::env::var("TMUX").is_ok() }`: One-liner tmux detection.
   - Include `#[cfg(test)] mod tests` with tests for `extract_jira_key`: "feature/UMP-1234-login" -> Some("UMP-1234"), "UMP-5678" -> Some("UMP-5678"), "main" -> None, "feature/no-ticket" -> None, "fix/UMP-9-short" -> Some("UMP-9").
