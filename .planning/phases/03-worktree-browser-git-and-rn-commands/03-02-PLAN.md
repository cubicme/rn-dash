<task type="auto">
  <name>Task 2: Create command_runner.rs and devices.rs, update infra/mod.rs</name>
  <files>src/infra/command_runner.rs, src/infra/devices.rs, src/infra/mod.rs</files>

1. Create `src/infra/command_runner.rs` following research Pattern 4:
   - `pub async fn spawn_command_task(spec: CommandSpec, worktree_path: PathBuf, current_branch: String, action_tx: UnboundedSender<Action>) -> tokio::task::JoinHandle<()>` — the single function for spawning any command.
   - Inside: calls `build_argv(&spec, &current_branch)` to get the full argv. First element is program, rest are args.
   - Spawns via `tokio::process::Command::new(program).args(args).current_dir(&worktree_path).stdout(Stdio::piped()).stderr(Stdio::piped()).kill_on_drop(true)`.
   - On spawn success: takes stdout + stderr, calls `stream_command_output()`, then waits for child to complete, then sends `Action::CommandExited`.
   - On spawn failure: sends `Action::CommandOutputLine(format!("[error] failed to spawn: {e}"))` then `Action::CommandExited`.
   - `async fn stream_command_output(stdout, stderr, tx)` — uses BufReader::new().lines() on both stdout and stderr, uses `tokio::select!` loop to read from both, sends each line as `Action::CommandOutputLine(l)`. Same pattern as stream_metro_logs in app.rs.
   - `fn build_argv(spec: &CommandSpec, current_branch: &str) -> Vec<String>` — for GitResetHard, returns `["git", "reset", "--hard", "origin/{current_branch}"]`. For all others, delegates to `spec.to_argv()`.

2. Create `src/infra/devices.rs` following research Pattern 9:
   - `pub fn parse_adb_devices(output: &str) -> Vec<DeviceInfo>` — skips header line "List of devices attached", splits remaining non-empty lines by tab, filters for state == "device", creates DeviceInfo with id=serial, name=serial.
   - `pub fn parse_xcrun_simctl(json_output: &str) -> Vec<DeviceInfo>` — parses JSON via serde_json::Value, iterates devices map values, filters isAvailable == true, creates DeviceInfo with id=udid, name="{name} ({state})".
   - `pub async fn list_android_devices() -> anyhow::Result<Vec<DeviceInfo>>` — runs `adb devices` via tokio::process::Command, parses output.
   - `pub async fn list_ios_devices() -> anyhow::Result<Vec<DeviceInfo>>` — runs `xcrun simctl list devices available --json` via tokio::process::Command, parses output.

3. Update `src/infra/mod.rs` — add:
