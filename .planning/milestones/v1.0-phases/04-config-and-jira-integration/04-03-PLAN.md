In src/ui/panels.rs, rewrite the item rendering closure inside render_worktree_list() (lines 35-74) to change the display order and styling:

1. **Branch name first (always visible):** After the metro badge span, add `Span::raw(&wt.branch)` as the primary text. This ensures the branch name is never truncated away.

2. **JIRA title or label second (visible color):** If `wt.label` is Some, show it after the branch name as ` - {label}` in `Color::Gray` (NOT DarkGray, NOT DIM). If `wt.label` is None but `wt.jira_title` is Some, show ` - {jira_title}` in `Color::Gray`. If neither is set, show nothing extra. Do NOT use parentheses -- use a dash separator for readability. Do NOT add the `Modifier::DIM` flag.

3. **Staleness icon:** Replace `" [stale]"` text with `" \u{26A0}"` (Unicode warning sign) keeping the Yellow color. This is compact and universally recognizable in terminals.

The span construction should be:
```rust
let mut spans = vec![
    Span::styled(badge_text, badge_style),
    Span::raw(&wt.branch),
];

// Secondary text: label takes priority over jira_title
if let Some(label) = &wt.label {
    spans.push(Span::styled(
        format!(" - {}", label),
        Style::default().fg(Color::Gray),
    ));
} else if let Some(title) = &wt.jira_title {
    spans.push(Span::styled(
        format!(" - {}", title),
        Style::default().fg(Color::Gray),
    ));
}

// Staleness icon
if wt.stale {
    spans.push(Span::styled(
        " \u{26A0}",
        Style::default().fg(Color::Yellow),
    ));
}
```
