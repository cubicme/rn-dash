## Accomplishments

- Replaced `tmux_available: bool` with `multiplexer: Option<Box<dyn Multiplexer>>` and added `claude_flags: String` and `config: Option<DashConfig>` to AppState
- Rewired `OpenClaudeCode` to use `detect_multiplexer()` in `spawn_blocking` with `claude_flags` from config — supports both tmux and zellij
- Replaced old 2-palette routing (Git+Rn stubs) with full 5-mode palette routing (Android, Ios, Clean, Sync, Git) with correct sub-keys
- Wired WorktreeTable to dispatch the 5 palette entry actions (a/i/x/s/g) plus C/L/f/!/Enter
- `EnterCleanPalette` in update() now opens `CleanToggle` modal immediately
- `CleanToggle` and `SyncBeforeRun` modals have full key interception in `handle_key()`
- f key dispatches `ToggleFullscreen` from WorktreeTable, MetroPane, and CommandOutput panels
- X (shift-x) dispatches `LogPanelClear` in MetroPane context

## Decisions Made

- `detect_multiplexer()` called inside `spawn_blocking` for OpenClaudeCode: avoids `Box<dyn Multiplexer>: Clone` requirement across async boundaries; re-detection is cheap (2 env var checks)
- `EnterCleanPalette` sets both `palette_mode` and `modal` simultaneously so the CleanToggle modal controls all key handling — the `PaletteMode::Clean` arm in palette routing is a fallback
- `X` (shift-x) for log clear in MetroPane context avoids collision with `x` (stop metro)
