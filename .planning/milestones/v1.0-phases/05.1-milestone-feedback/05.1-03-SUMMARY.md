## What Was Built

### CommandSpec Extensions (5 new variants)
- `GitFetch` — `git fetch --all --tags`
- `GitResetHardFetch` — fetch then reset to origin/branch (destructive)
- `RnReleaseBuild` — `gradlew assembleRelease`
- `AdbInstallApk` — `adb install -r` the release APK
- `ShellCommand { command }` — `sh -c <command>` (needs_text_input)

### CleanOptions Struct
Four booleans: `node_modules`, `pods`, `android`, `sync_after`. Derives Default so all fields start false.

### ModalState Extensions
- `CleanToggle { options: CleanOptions }` — clean submenu with toggleable targets
- `SyncBeforeRun { run_command, needs_pods }` — stale-worktree prompt before run commands

### PaletteMode (replaced Git/Rn with 5-variant scheme)
`Android`, `Ios`, `Clean`, `Sync`, `Git` — matches the Phase 05.1 command scheme from CONTEXT.md.

### Action Variants (15 new)
Submenu activation (EnterAndroidPalette, EnterIosPalette, EnterCleanPalette, EnterSyncPalette), clean toggles (4), ToggleFullscreen, StartShellCommand, SimulatorUsed(String), SyncBeforeRunAccept, SyncBeforeRunDecline, LogPanelClear.

`LogPanelClear` is fully wired — clears `metro_logs` and resets `log_scroll_offset`.

### Infrastructure Utilities
- `check_stale_pods(path)` — checks `ios/Pods/` mtime vs `ios/Podfile.lock` mtime
- `sim_history.rs` — `load_sim_history()` and `record_sim_used(udid)`, persists to `~/.config/ump-dash/sim_history.json` as JSON array, max 20 entries, most-recent-first
