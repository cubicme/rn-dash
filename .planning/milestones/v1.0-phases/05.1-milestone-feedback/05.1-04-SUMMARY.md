# Phase 5.1 Plan 04: Layout Restructure and Worktree Table Summary

Replaced the worktree list widget with a structured Table widget at the bottom of the layout. Metro and output panes moved to the top section. Added fullscreen mode for metro/output panes.

## What Was Built

### Task 1: AppState TableState + fullscreen_panel + FocusedPanel rename

- `worktree_list_state: ListState` removed from AppState, replaced with `worktree_table_state: TableState`
- `selected_worktree_id: Option<WorktreeId>` added — stable selection identity that survives sorts
- `fullscreen_panel: Option<FocusedPanel>` added — controls single-pane mode
- `FocusedPanel::WorktreeList` renamed to `FocusedPanel::WorktreeTable` throughout app.rs, panels.rs, footer.rs
- `ToggleFullscreen` action wired: toggles `fullscreen_panel` for MetroPane/CommandOutput only
- Metro-active worktree pinned to index 0 in `WorktreesLoaded` before re-deriving selected index
- `WorktreeSelectNext/Prev` now also update `selected_worktree_id` for stable tracking

### Task 2: Layout restructure + render_worktree_table

- `src/ui/mod.rs` rewritten: vertical layout top (metro+output) | bottom (table) | footer
- Fullscreen mode: single panel + footer, overlays still render on top
- Log panel variant: left-top metro, left-bottom log, right output
- `render_worktree_list` removed entirely from panels.rs
- `render_worktree_table` added with 6 columns: LABEL, WORKTREE, BRANCH, TICKET, TITLE, icons
- `truncate()` helper trims strings with "..." on narrow terminals
- Metro-active row rendered bold; icons column shows `●` for running and `⚠` for stale
- Columns use `Constraint::Min` for flexible truncation and `Constraint::Length` for fixed widths
- `render_stateful_widget` wired to `state.worktree_table_state` directly
