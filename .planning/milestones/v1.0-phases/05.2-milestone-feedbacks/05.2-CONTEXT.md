<decisions>
## Implementation Decisions

### External Metro Conflict
- Check port 8081 on startup AND before every metro start
- When external metro detected, show prompt with PID + working directory (via lsof/ps)
- Prompt offers two actions: "Kill it" or "Cancel"
- If user kills: auto-start metro in the currently selected worktree (seamless flow)

### Pane Visual Style
- All panes get double-line borders (╔═══╗ style) — oldschool terminal aesthetic
- Pane titles embedded top-left inside the border (e.g., ╔═ Metro — running (feature-branch) ═══╗)
- Metro pane title is dynamic: "● Metro — running (worktree-name)" or "○ Metro — stopped"
- Output pane title shows queue state (e.g., "Output (2 queued)")
- Worktree table also gets titled border ("Worktrees")
- Remove standalone "RUNNING [ump]" text — that info now lives in the metro pane title

### Table Column Layout
- Column order: Status | Label | Branch | Ticket
- Rename "Title" column to "Ticket" — drop separate ticket number column
- Branch column should be half its current size
- Ticket column gets the most space (largest column)

### Data Dependency Model
- Explicit domain mapping: a function like `refresh_needed(cmd) -> RefreshSet` in domain/ that returns what to refresh after a command completes
- Single source of truth, testable — not scattered across update() match arms
- Branch-changing git ops (checkout, checkout -b, rebase, reset-hard) trigger full worktree reload
- Non-branch-changing git ops (pull, push, fetch) do NOT trigger worktree reload
- Install/clean commands (yarn install, pod-install, clean variants) trigger staleness refresh
- JIRA title fetch triggered when worktree reload detects new/changed branch names

### Staleness Detection Fix
- Current staleness detection is unreliable — both false positives and false negatives
- Needs investigation and rewrite of `check_stale()` logic in infra/worktrees.rs
- Must be included in UAT with explicit test cases for stale and non-stale scenarios
- Staleness refreshes after install/clean commands complete (not just on worktree reload)

### Scrolling & Navigation
- Auto-scroll in metro log pane (follow tail, like a real terminal)
- hjkl, G, gg work everywhere for scrolling — these are universal, not shown in footer hints
- Scroll support in metro pane and output pane (j/k to scroll, G for bottom, gg for top)

### Fullscreen Behavior
- Pressing Tab while in fullscreen exits fullscreen (returns to normal layout)

### Bug Fixes (no discussion needed — straightforward)
- iOS device picker (i>d) shows simulators instead of physical devices — fix device enumeration filtering
- Metro-active worktree not visually highlighted in table — fix row styling (was decided in 05.1: bold/background + ● icon)
- Metro debugger (j key) not working — fix stdin forwarding or action dispatch
- Metro log auto-scroll not working — implement tail-follow behavior

### Claude's Discretion
- Exact RefreshSet type design (bitflags, enum set, or simple struct)
- Metro pane title format details (exact separator, truncation of long worktree names)
- How to detect external metro PID and working directory (lsof vs /proc vs ps)
- Scroll position management when new content arrives during manual scroll

</decisions>
