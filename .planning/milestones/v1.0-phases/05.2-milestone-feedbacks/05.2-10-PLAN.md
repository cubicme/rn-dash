```rust
/// Returns the preferred display prefix for this worktree, in priority order:
/// 1. Custom label (user-assigned)
/// 2. JIRA ticket key (e.g., "UMP-1234") — short identifier, not full title
/// 3. Branch name
/// 4. Workspace directory name (fallback)
///
/// Used as the single source of truth for naming: Claude tab name, pane title, etc.
pub fn preferred_prefix(&self) -> String {
    if let Some(label) = &self.label {
        return label.clone();
    }
    // Extract ticket key from branch (short, like "UMP-1234")
    if let Some(key) = crate::infra::jira::extract_jira_key(&self.branch) {
        return key;
    }
    // Branch name
    if !self.branch.is_empty() && self.branch != "(unknown)" {
        return self.branch.clone();
    }
    // Fallback to directory basename
    self.path.file_name()
        .and_then(|n| n.to_str())
        .unwrap_or("worktree")
        .to_string()
}
```

Note: This returns a ticket KEY (like "UMP-1234"), not the full title. The key is short enough for tab names and pane titles. The full title stays in the table's Ticket column.

Important: `extract_jira_key` is in infra — calling it from domain violates ARCH-01 strictly. To keep domain pure, pass extracted key as a method parameter OR store the extracted key on the Worktree struct. The simplest approach: add `pub jira_key: Option<String>` to Worktree, set it during parse/WorktreesLoaded, and check it in preferred_prefix(). This avoids the infra dependency.

let title = match &state.metro.status {
    MetroStatus::Running { pid: _, worktree_id } => {
        // Look up the actual worktree to get the preferred display name
        let wt_display = state.worktrees.iter()
            .find(|wt| {
                wt.path.file_name()
                    .and_then(|n| n.to_str())
                    .unwrap_or("") == worktree_id

    MetroStatus::Stopped => " Metro -- stopped ".to_string(),
