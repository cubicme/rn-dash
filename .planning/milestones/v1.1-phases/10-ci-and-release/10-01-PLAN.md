  <action>
Create `.github/workflows/release.yml` with the following specification:

**Trigger:** `push` with tag filter `v*` (e.g., v0.1.0, v1.0.0).

**name:** `Release`

**Permissions:** `contents: write` (needed to create releases and upload assets).

**Job: build**
- Matrix:
  - `{ os: macos-latest, target: aarch64-apple-darwin }` (Apple Silicon)
  - `{ os: macos-latest, target: x86_64-apple-darwin }` (Intel Mac)
  - `{ os: ubuntu-latest, target: x86_64-unknown-linux-gnu }` (Linux)

- Steps:
  1. `actions/checkout@v4`
  2. `dtolnay/rust-toolchain@stable` with `targets: ${{ matrix.target }}`
  3. `Swatinem/rust-cache@v2` with `key: ${{ matrix.target }}`
  4. Build: `cargo build --release --target ${{ matrix.target }}`
  5. Package: Create a tar.gz archive named `rn-dash-${{ matrix.target }}.tar.gz` containing the binary from `target/${{ matrix.target }}/release/rn-dash`. Use `tar -czf` in the repo root.
  6. Upload artifact: `actions/upload-artifact@v4` with name `rn-dash-${{ matrix.target }}` and path to the tar.gz file.

**Job: release**
- `needs: build`
- `runs-on: ubuntu-latest`
- Steps:
  1. Download all artifacts: `actions/download-artifact@v4` with `merge-multiple: true` (downloads all uploaded tarballs into one directory)
  2. Create GitHub Release: Use `softprops/action-gh-release@v2` with:
     - `files: rn-dash-*.tar.gz`
     - `generate_release_notes: true`
     - `tag_name: ${{ github.ref_name }}`

Use `CARGO_TERM_COLOR: always` as env at workflow level.
  </action>

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner | Workflow executes on GitHub-hosted runners with access to repo contents |
| Tag push trigger | Anyone with push access can trigger release by pushing a tag |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | release.yml | accept | Only repo collaborators can push tags; GitHub branch protection can restrict tag creation if needed |
| T-10-02 | Elevation | release.yml | mitigate | Scope permissions to `contents: write` only (not `permissions: write-all`); release job only runs after build succeeds |
| T-10-03 | Spoofing | ci.yml | accept | Uses pinned major versions of trusted actions (actions/*, dtolnay/*, Swatinem/*); acceptable risk for open-source tooling |
</threat_model>
